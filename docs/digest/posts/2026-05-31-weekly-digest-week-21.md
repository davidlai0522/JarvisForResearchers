## Papers This Week

### [Integrable Elasticity via Neural Demand Potentials](https://arxiv.org/abs/2605.22820)
*The Integrable Context-Dependent Demand Network (ICDN) is a demand-first neural model that represents log-demand as a smooth, context-conditioned function of log-prices, allowing elasticities to be derived exactly as derivatives of the learned demand surface.*
ICDN decomposes price responses into linear own-price effects, directed cross-price interactions, and spline-based nonlinear refinements. It computes own- and cross-price elasticities by exploiting analytic derivatives of product-specific spline bases, avoiding dense Jacobian or Hessian evaluations.
**Why it matters:** ICDN provides a framework to derive elasticities exactly from a single, smooth demand surface, enhancing analytical tractability.

### [SkillOpt: Executive Strategy for Self-Evolving Agent Skills](https://arxiv.org/abs/2605.23904)
*SkillOpt introduces a systematic, controllable text-space optimizer that treats agent skills as an external state, allowing for deep-learning-style optimization of procedural knowledge without modifying the frozen target model weights.*
The framework uses rollout batches, reflection minibatches, and textual learning rates to optimize skills. Across its empirical study, SkillOpt achieved an average gain of +23.5 points over no skill when using GPT-5.5 in a direct chat setting.
**Why it matters:** SkillOpt provides a framework to systematically optimize agent procedures (skills) using deep-learning optimization principles without requiring access to model weights.

### [TriSplat: Simulation-Ready Feed-Forward 3D Scene Reconstruction](https://arxiv.org/abs/2605.26115)
*TriSplat is a feed-forward 3D scene reconstruction network that uses oriented triangle primitives to directly export simulation-ready mesh scenes from sparse, unposed images in a single forward pass.*
The network jointly predicts geometry, appearance, and camera poses using oriented triangle primitives. This triangle-native representation eliminates post-hoc mesh extraction, allowing direct consumption by physics engines.
**Why it matters:** For simulation-ready 3D reconstruction, using explicit surface primitives (like triangles) natively in a feed-forward pipeline avoids costly post-processing steps.

### [Latent Action Diffusion for Cross-Embodiment Manipulation](https://arxiv.org/abs/2506.14608)
*This paper introduces Latent Action Diffusion to unify diverse end-effector action spaces into a semantically aligned latent space, enabling cross-embodiment policy learning for multi-robot control and skill transfer.*
The method uses contrastive learning to unify diverse action spaces into a single latent space. Factorizing policies into a latent, embodiment-agnostic policy and embodiment-specific action decoders resulted in up to a 25.3% improvement in manipulation success rate over single-embodiment policies.
**Why it matters:** Use contrastive learning to bridge heterogeneous action spaces when transferring skills between robots with different end-effectors.

### [Algorithmic Monocultures in Hiring](https://arxiv.org/abs/2605.27371)
*This study investigates algorithmic monoculture in hiring by analyzing a novel dataset from the vendor pymetrics, finding clear racial disparities and systemic rejection patterns when examining hiring decisions on a per-position basis.*
The research provided evidence of systemic rejections at scale due to deployed hiring algorithms. Specifically, the adverse impact against Black applicants per position was measured at 10.62%.
**Why it matters:** Analyzing hiring algorithms on a per-position basis is critical when assessing vendor-mediated systems, as aggregate metrics can mask localized discrimination.

### [Physics Is All You Need? A Case Study in Physicist-Supervised AI Development of Scientific Software](https://arxiv.org/abs/2605.30353)
*This case study details how a physicist supervised an AI coding agent (Claude Code, Sonnet, and Opus) over 12 work days and 57 sessions to build a differentiable one-loop perturbation theory module (CLAX-PT $\S$), demonstrating that the supervision protocol, not model capability, was key to achieving trustworthy scientific software.*
The study identified that the supervision protocol was the primary factor in the agent's output trustworthiness. The agent achieved $\pm 1\%$ accuracy against the established CLASS-PT benchmark.
**Why it matters:** For scientific AI development, the supervision protocol is as critical as the model's raw capability.
